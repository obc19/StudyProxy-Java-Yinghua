
### 3.2 并发控制

| 层级 | 最大并发 | 实现方式 |
|------|----------|----------|
| 账号级 | 100 | `Executors.newFixedThreadPool(100)` |
| 课程级 | 5/账号 | `Executors.newFixedThreadPool(5)` |
| 视频级 | 串行 | 同步循环提交 |

### 3.3 线程安全

- **YinghuaUserCache**: 每个账号独立实例，无线程安全问题
- **CountDownLatch**: 用于等待所有任务完成
- **无共享状态**: 各账号/课程独立处理，无锁竞争

## 5. 错误处理

### 5.1 错误类型

| 错误类型 | 处理方式 | 代码位置 |
|----------|----------|----------|
| 账号密码错误 | System.exit(1) | YinghuaLoginAction.isPasswordError() |
| 验证码识别失败 | 重试(最多10次) | YinghuaLoginAction.login() |
| 网络请求失败 | 重试(最多3次) | YinghuaLoginApi.executeWithRetry() |
| 视频锁定 | 跳过 | YinghuaVideoStudyService.studyVideo() |
| 考试答题失败 | 记录错误 | YinghuaVideoStudyService.handleExam() |

### 5.2 重试机制

```java
// 登录重试 (最多10次)
for (int i = 0; i < MAX_RETRY; i++) {
    try {
        // 登录逻辑
    } catch (Exception e) {
        // 重试
    }
}

// API重试 (最多3次)
private String executeWithRetry(Callable<String> task, int maxRetry) {
    for (int i = 0; i < maxRetry; i++) {
        try {
            return task.call();
        } catch (Exception e) {
            if (i == maxRetry - 1) throw e;
        }
    }
}
```
